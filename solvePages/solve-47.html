Problem Statement

Write a C function named **my_memcpy** that mimics the functionality of the standard library function `memcpy`. The function must copy $N$ bytes of data from a memory area pointed to by a `const void*` source to a memory area pointed to by a `void*` destination. Using **`void*` is essential** for handling any data type.

Input Format

The program should hardcode an array of integers and copy it to a new array using `my_memcpy`.

Output Format

Print the elements of the destination array to confirm the copy was successful, and also print the original source data for comparison.

Examples:

Example 1
Input Array
Source Array: {10, 20, 30, 40, 50}
Output
Source data: 10 20 30 40 50 Destination data after my_memcpy: 10 20 30 40 50
Constraints:
  • The function prototype must use `void*` for the pointers and `size_t` for the byte count.
  • The implementation must perform the copy byte-by-byte by casting the `void*` pointers to `char*`.
  • The example usage must demonstrate copying an array of integers, requiring correct calculation of the total bytes to copy.

Hints:

1

The crucial step is to cast both `void*` arguments to `char*` to allow for pointer arithmetic and access to individual bytes.

2

The number of bytes to copy for an integer array is `sizeof(source_array)`. This is equal to `num_elements * sizeof(int)`.

3

A simple `for` loop that iterates from 0 up to `n` (the byte count) can be used to copy `d[i] = s[i]`, where `d` and `s` are the `char*` pointers.

Explanation:

The core of a custom `memcpy` lies in using `void*` pointers for generality and casting them to `char*` for byte-level control. This is necessary because `void*` cannot be dereferenced or used for pointer arithmetic.

The `dest` and `src` pointers are cast to `char*` pointers (`d` and `s`). Since a `char` is guaranteed to be one byte in C, this allows us to address and copy individual bytes.
A loop iterates `n` times, where `n` is the total number of bytes to copy. In each iteration, `d[i] = s[i]` copies one byte from the source memory location to the destination memory location.
The main function demonstrates usage by dynamically allocating memory for the destination array (`dest_arr`) and calculating the byte count using `sizeof(source_arr)`.

#include <stdio.h> #include <string.h> #include <stdlib.h> // Custom implementation of memcpy void* my_memcpy(void *dest, const void *src, size_t n) { // 1. Cast the void pointers to char pointers for byte-level access char *d = (char *)dest; const char *s = (const char *)src; // 2. Iterate and copy byte by byte for (size_t i = 0; i < n; i++) { d[i] = s[i]; // Copy the byte at index i } // Return the starting address of the destination return dest; } void printArray(const char* label, int arr[], int size) { printf("%s", label); for (int i = 0; i < size; i++) { printf("%d ", arr[i]); } printf("\\n"); } int main() { int source_arr[] = {10, 20, 30, 40, 50}; int n = sizeof(source_arr) / sizeof(source_arr[0]); // Allocate memory for the destination array int *dest_arr = (int*)malloc(sizeof(source_arr)); if (dest_arr == NULL) { printf("Memory allocation failed!\\n"); return 1; } // The number of bytes to copy is the total size of the source array size_t bytes_to_copy = sizeof(source_arr); // --- Perform the copy using the custom function --- my_memcpy(dest_arr, source_arr, bytes_to_copy); // Print source and destination data printArray("Source data: ", source_arr, n); printArray("Destination data after my_memcpy: ", dest_arr, n); free(dest_arr); return 0; }
