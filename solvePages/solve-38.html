Problem Statement

Write a C program to solve the **Josephus Problem**. Given $N$ people standing in a circle and a counting step $K$, starting from the first person, every $K$-th person is eliminated until only one person remains. The program should return the position (1-based index) of the last person remaining (the survivor). Use a dynamic array or linked list structure to simulate the game flow. The provided solution uses **dynamic array simulation**.

Input Format

Use hardcoded integers $N$ (number of people) and $K$ (counting step).

Output Format

Print the position of the survivor.

Examples:

Example 1
Input (Hardcoded)
N=7 people, K=3 step
Output
The survivor is at position 4
Constraints:
  • Must handle $N$ up to 1000 and $K$ up to 100 (for simulation).
  • The solution should correctly model the circular elimination process.
  • Must use 1-based indexing for the people in the array (e.g., person 1, 2, ..., N).
  • The output must be the final position (1-based index).

Hints:

1

The next person to be eliminated can be found using the modulo operator: next\_index = (current\_index + K - 1) % remaining\_people.

2

When simulating array removal, shift all subsequent elements one position to the left and decrement the count of remaining people.

3

Remember to free() the dynamically allocated array after the survivor is found.

Explanation:

This solution uses a dynamic array to store the list of remaining people and simulates the elimination process step-by-step. While slower than the mathematical recurrence relation, it provides a direct simulation as requested.

The array `people` is initialized with values $1$ to $N$. An index `current_index` tracks the starting point of the count.
The loop continues as long as `remaining` > 1. The index of the person to be removed is calculated using (current_index + K - 1) % remaining.
Removal is simulated by shifting elements: a loop moves $people[i+1]$ to $people[i]$ starting from the `current_index`. `remaining` is decremented.
Crucially, the `current_index` is updated using current_index % remaining because the person *after* the eliminated person becomes the new starting point for the count, which corresponds to the current index position if it wasn't the last element.

#include <stdio.h> #include <stdlib.h> // Function to solve the Josephus Problem using array simulation int josephus(int n, int k) { // Use a list/array to hold the person positions (1 to n) int *people = (int*)malloc(n * sizeof(int)); if (people == NULL) return -1; for (int i = 0; i < n; i++) { people[i] = i + 1; // 1-based indexing for people } int current_index = 0; // The index of the person starting the count (0-based) int remaining = n; // Loop until only one person remains while (remaining > 1) { // Calculate the index of the person to be eliminated // (k-1) because we are counting k steps from the current_index (0-based) current_index = (current_index + k - 1) % remaining; // Simulate elimination by shifting all subsequent elements left. for (int i = current_index; i < remaining - 1; i++) { people[i] = people[i + 1]; } remaining--; // One person is eliminated // The next count starts from the person who moved into the current_index slot. // If the eliminated person was the last element, the new starting index wraps to 0. current_index = current_index % remaining; } int survivor = people[0]; free(people); return survivor; } int main() { int N = 7; // Number of people int K = 3; // Step int survivor_pos = josephus(N, K); if (survivor_pos != -1) { printf("The survivor is at position %d\\n", survivor_pos); } else { printf("Error in calculation.\\n"); } return 0; }

This method explicitly simulates the circle by removing elements and shifting the rest of the array. The removal process takes $O(N)$ time per elimination, and since there are $N-1$ eliminations, the total time complexity is $O(N^2)$.
